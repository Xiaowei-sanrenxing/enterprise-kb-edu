# 任务卡 · 续费复购转介绍（renewal-referral）

> Agent 操作总台。接到"续班推进/复购/老带新/口碑裂变"类任务，从这里开始。

## 🎯 目标(Goal)
把已交付的学员，转化为续费/复购 + 主动转介绍——**让交付成果变成下一轮获客的起点**（老带新成本远低于公域获客）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/续费节点打法.md`](context/续费节点打法.md) — 在学员"体验到价值"的节点推续费，不是临到期才催
2. [`context/转介绍裂变机制.md`](context/转介绍裂变机制.md) — 老带新的触发点、激励、话术
3. 〔学员成果记录〕 — 客户补：来自交付工作流（脱敏）
4. 〔续费/转介绍激励政策〕 — 客户补：当期政策

## 🪜 指令(Instructions) —— 标准动作流
1. **识别价值节点**：学员拿到成果/做出效果的时刻 = 最佳续费/转介绍窗口
2. **续费**：在价值节点而非到期日推进，用成果说话，不靠催
3. **复购**：根据学员阶段推荐进阶/关联课程
4. **转介绍**：在学员满意峰值发起老带新，给清晰激励 + 现成转介绍话术/物料
5. **闭环**：转介绍来的新线索 → 交回销转工作流（高信任度，转化率更高）
6. 学员成果引用对外 → 必脱敏（走交付工作流的脱敏铁律）

## 🛠 工具(Tools)
〔续费提醒（按价值节点）〕 · 〔转介绍激励系统〕 · 成果记录检索 · 转介绍话术库

## 🔒 权限(Permission)
- 可读：续费政策、转介绍话术、（脱敏的）成果记录
- 🔴 受限：学员个人信息/联系方式
- 需人工确认：超标准的续费优惠、转介绍激励发放

## ✅ 验收(Acceptance)
- 续费在价值节点推进（非到期催）｜ 转介绍在满意峰值发起 ｜ 老带新线索回流销转
- 详见 evals.md

## 🔄 自我进化接口
记录"哪个价值节点续费/转介绍成功率最高" → 回流续费节点打法 → 优化触达时机。

# 续费节点打法

> 核心：**在学员"体验到价值"的节点推续费，不是临到期才催。**

## 为什么不靠催
到期日催续费 = 把决策压在"还想不想花钱"上，抗拒最强。价值节点推 = 把决策建在"我已经получ得到了"上，顺势而为。

## 价值节点（按这些时机推，客户填自己课程的具体节点）
| 节点 | 信号 | 动作 |
|---|---|---|
| 首次出成果 | 学员做出第一个可见效果 | 肯定成果 → 自然引出进阶 |
| 阶段达成 | 完成一个里程碑 | 复盘收获 → 推下一阶段 |
| 高满意时刻 | 主动表达感谢/认可 | 满意峰值 → 续费 + 转介绍双触发 |
| 临到期 | 课程将结束 | 兜底动作，但不应是唯一动作 |

## 用法
盯学员的**价值节点**而非日历。节点到了，用成果说话推续费/进阶。

# 转介绍裂变机制

> 让交付成果变成下一轮获客起点。老带新成本远低于公域获客，且信任度天然高。

## 三要素（缺一不可）
1. **时机**：在学员**满意峰值**发起（出成果/主动认可时），不是随机群发。
2. **激励**：给清晰、即时的激励（双向更好：推荐人 + 被推荐人都得益）。
3. **弹药**：给现成的转介绍话术/物料，让学员"转发即可"，零创作成本。

## 闭环
转介绍来的新线索 → 交回 **销转工作流（sales-conversion）**。这类线索带着老学员背书，信任度高，转化率显著高于冷流量。

## 触发点示例（客户填自己的）
- 学员晒成果 → "要不要把这个方法分享给同样需要的朋友？我给你份现成的"
- 满意反馈 → 顺势给转介绍激励 + 物料

## 用法
别等学员"想起来"转介绍。在满意峰值主动发起，备好激励和弹药。

# 权限分级 · 四级

> 企业库和个人库最硬的区别。没有权限治理 = AI 资料垃圾场 + 隐私事故。

| 级别 | 含义 | 典型资产 | agent 行为 |
|---|---|---|---|
| 🟢 公开 | 可对外发布 | 对外物料、方法论、介绍 | 可写入对外产物 |
| 🟡 内部 | 团队内用 | 话术、SOP、运营数据 | 内部可用,不对外 |
| 🔴 受限 | 隐私/敏感 | 客户对话、通话记录、手机号、名单 | **绝不写入对外产物,只内部分析** |
| ⛔ 禁止入库 | 不该进库 | 证件、银行信息、未授权第三方资料 | 拒绝入库 |

## agent 红线
1. 🔴 受限资产 → 输出时必须脱敏/打码,绝不原文外发
2. 超标准的报价/优惠/对外承诺 → 必须人工确认
3. 不确定权限 → 默认按更高一级处理,问用户


## 治理 · 行业合规默认.md

# 教培行业 · 合规与权限默认值

> 合成进成品库 `90-governance/行业合规默认.md`。客户可按自身情况收紧。
> ⚠️ 教培是强监管行业，对外宣传/收费/排课前必读本页。各地细则可能更严，以当地教育/市监要求为准。

## 默认权限分级（通用四级 + 教培补充）
| 资产类型 | 默认级别 |
|---|---|
| 学员/家长个人信息（姓名/手机/微信/成绩/档案）| 🔴 受限（永不对外、输出必脱敏改化名）|
| 未成年人相关信息 | 🔴 受限（更严，去任何可定位个人的细节）|
| 内部话术/SOP/课件/运营数据 | 🟡 内部 |
| 对外招生物料/科普/品牌介绍 | 🟢 公开（须经合规审核，见下方红线）|
| 办学许可/教师资质证照 | ⛔ 禁止入库（证件类，留原地索引）|

## 教培行业红线（agent 必须遵守，对外产出前逐条过）
1. **禁虚假宣传**：不得承诺/暗示"保过、保升学、保提分 X 名"，不得虚构或夸大升学率、师资、获奖、效果。广告禁绝对化用语（最好/第一/100%）。
2. **收费合规**：学科类一次性收费跨度不得超过 **3 个月或 60 课时**（取其短），不得超期预收；遵守预付费资金监管要求。
3. **双减约束（学科类）**：不得占用国家法定节假日、休息日及寒暑假组织学科类培训；排课/招生文案不得违反。
4. **教学合规**：不得超前超纲教学；课程承诺与实际一致。
5. **资质公示**：学科类教师须具备相应教师资格并依规公示；对外不得虚标师资。
6. **未成年人保护**：涉及未成年学员的信息与肖像，未经监护人授权不得用于对外宣传。

## agent 行为
- 任何对外招生/宣传产出（发公众号、给渠道、写海报文案）→ **先过上面 6 条红线**，命中任一 → 改写或人工确认。
- 涉及收费周期、升学承诺、节假日排课 → **必须人工确认**，agent 不得自行拍板。
- 不确定是否合规 → 默认按更严处理，问负责人。

> 合规是教培的生命线。一条虚假宣传可能导致整个机构被处罚——对外内容宁可保守。
