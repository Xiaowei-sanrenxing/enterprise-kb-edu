# 从这里开始 · 〔机构名〕 企业 AI 知识库（教培版）

> 这个库是给 agent 使用的执行底座，不是要求每个人手动翻文件、手动跑 SOP。
> 首次使用时，先让 agent 问清你的角色，再进入对应工作流或维护路径。

---

## 第 0 步 · 复制这句话给 AI

新用户不需要先理解 `AGENTS.md`、`INDEX.md`、`_task-card.md` 或工作流结构。第一次打开这个知识库，只要把下面这句话发给 AI：

```text
帮我开始使用这个知识库。
```

AI 看到这句话后，应该自动读取 `START-HERE.md`、`AGENTS.md` 和 `INDEX.md`，然后先问：

```text
你现在使用这个知识库的角色是什么？

1. 老板/负责人：看全局、定方向、定边界、审核定版
2. DRI/部门主管：维护某条工作流，审核 agent 输出
3. 普通员工：把具体任务交给 agent，补充事实和素材
4. 知识库维护者：补资产、管权限、做定版和同步
5. agent：按 AGENTS.md 路由并执行任务

你这次想完成什么事？
```

如果 AI 没有主动读取知识库，再补一句：

```text
请先读 START-HERE.md、AGENTS.md 和 INDEX.md，再带我开始。
```

## 按角色进入

### 老板/负责人

你不需要手动跑每条 SOP。你主要让 agent 做全局体检、缺口盘点、边界确认和定版审核。

```text
读 AGENTS.md、INDEX.md 和 00-org/dri-map.md，帮我检查这套知识库当前还有哪些关键缺口，按优先级列出来。
```

### DRI/部门主管

你负责某条工作流的业务正确性和资产维护。让 agent 读你负责的任务卡和 context，再帮你补资产、产出草稿、做复盘。

```text
读 <workflow> 的 _task-card.md 和 context，基于我下面提供的素材，帮我更新草稿资产，并标出哪些内容需要人工定版。
```

### 普通员工

你不需要自己翻完整知识库。你只要把具体任务、已知事实、素材和限制说清楚，让 agent 判断走哪条工作流并执行。

```text
读 AGENTS.md，判断这件事该走哪条工作流。
我的任务是：<具体任务>
已知信息是：<事实/素材>
请输出：<话术/表格/方案/复盘/清单>
```

### 知识库维护者

你负责让库持续变好：补素材、检查权限、更新索引、推动草稿定版和同步。

```text
读 AGENTS.md、INDEX.md 和 90-governance，帮我检查当前知识库有哪些草稿、占位符、权限风险和待同步资产。
```

### Agent

agent 不应该让用户自己拼上下文。接到任务后，按 `AGENTS.md` 判断工作流，读取任务卡、context、权限和定版规则，再执行并自检。

## 当前预置工作流

本模板预置 5 条教培行业常见工作流：

- **获客引流**（`traffic-acquisition`）— 把潜在家长/学员从公域引到私域或到店
- **销售转化**（`sales-conversion`）— 把线索从首触推进到报名成交或明确流失
- **课程研发**（`course-development`）— 课程设计、课件讲义、迭代更新
- **教学交付**（`teaching-delivery`）— 排课上课、作业答疑、学员陪跑
- **续费复购转介绍**（`renewal-referral`）— 续班、口碑裂变、老带新

如果某条工作流不适合你们公司，可以删掉；如果你们有特有工作流，可以复制一条现有工作流目录后改造。

## 之后边用边长

- 新料 -> 按 `90-governance/intake-rules.md` 入库
- 草稿 -> 经 DRI 或负责人审核后定版
- 受限资料 -> 留在受限区域，不进对外产物
- 精华定版后 -> 按 `90-governance/feishu-sync.md` 同步给团队
- 踩坑、客户反馈、新情况 -> 记录到 `90-governance/进化日志.md`

---

## 三条绝对别碰

1. 别在母版上直接干某一单/某一期的活。具体单子或班期在自己的项目里做，库里只沉淀可复用的标准。
2. 客户隐私（真名/手机/微信）永不进 `定版/`、永不对外。
3. `定版/` 里的东西要改，先经过负责人把关。

> 有问题就问 AI："读 AGENTS.md，判断这件事该走哪条工作流，并告诉我需要补哪些事实。"
