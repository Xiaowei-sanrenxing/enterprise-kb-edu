# 任务卡 · 课程研发（course-development）

> Agent 操作总台。接到"设计课程/写大纲/对稿/做课件/迭代改进"类任务，从这里开始。

## 🎯 目标(Goal)
把一个课程主题，从立项推进到可交付的课件/讲义，**且把上一期的问题修掉**——研发不是从零，是站在上期复盘上迭代。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/课程研发五阶段流程.md`](context/课程研发五阶段流程.md) — 立项→大纲→对稿→课件→迭代
2. [`context/上期问题修复机制.md`](context/上期问题修复机制.md) — 研发前先读上期问题对照表，逐条修
3. 〔本期课程总议程〕 — 客户补：当期议程
4. 〔讲师对稿规范〕 — 客户补：多讲师协作对稿口径

## 🪜 指令(Instructions) —— 标准动作流
1. **立项**：明确本期主题、目标学员、要解决的核心问题
2. **读上期问题对照表**：把上期暴露的问题列成"本期必修清单"
3. **大纲**：各讲 outline，标清每讲目标 + 带走物（学员能带走的工具/模板）
4. **对稿**：多讲师按统一口径对齐，避免内容重叠/冲突
5. **课件**：按大纲产出课件/讲义，草稿进 `_草稿区/`
6. **迭代**：课程跑完→收集学员反馈→进上期问题对照表，供下期
7. 定稿后由负责人提名进 `定版/`（补 frontmatter）

## 🛠 工具(Tools)
〔课件模板〕 · 〔对稿协作文档〕 · 大纲检查清单 · 带走物设计模板

## 🔒 权限(Permission)
- 可读：议程、大纲、课件、上期问题表
- 🟡 内部：未发布课件 → 不对外
- 需人工确认：课程定版、对外宣称的课程承诺/效果

## ✅ 验收(Acceptance)
- 大纲每讲有明确目标 + 带走物 ｜ 上期问题逐条有修复动作 ｜ 多讲师无内容冲突
- 详见 evals.md

## 🔄 自我进化接口
每期课程收集学员反馈 + 答疑高频问题 → 回流上期问题对照表 → 下期研发默认带上。
