# 任务卡 · 私域销转（sales-conversion）· 脱敏示范

> Agent 操作总台。接到"接待线索/聊天/推课/反对处理/报价/挽回"类任务,从这里开始。

## 🎯 目标(Goal)
把一个新线索,从首次触达推进到成交或明确流失原因,全程不踩三大坑(响应慢/开场模板化/发料就停)。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/客户分级标准-A到F.md`](context/客户分级标准-A到F.md) — 30 秒判级,决定打法
2. [`context/8阶段销转SOP.md`](context/8阶段销转SOP.md) — 阶段动作 + 漏斗卡点
3. 〔话术库索引〕 — 按场景查标准话术(真实库含此项)
4. 〔转化诊断数据〕 — 必须避开的坑(真实库含此项)

## 🪜 指令(Instructions)
1. 5 秒判来源 → 初判等级
2. 5 分钟内首回,开放式开场(禁"在吗"/二选一)
3. 收集画像 → 锁定等级
4. 按等级走打法
5. 精准发料 → 必追问(禁发完就停)
6. 多跳追问 → 推进深度对话
7. 推课 → 处理反对 → 报价 → 成交/记录流失

## 🛠 工具(Tools)
〔CRM/客户档案读写〕 · 话术库检索 · 〔诊断简报模板〕

## 🔒 权限(Permission)
- 可读:话术、SOP、分级、诊断
- 🔴 受限:客户聊天原文/手机号/微信号 → 不得写入任何对外产物
- 需人工确认:超标准的报价优惠、大客户移交

## ✅ 验收(Acceptance)
- 首回 < 5 分钟 ｜ 开场开放式 ｜ 发料后有追问 ｜ 流失有记录原因
- 详见 evals.md(真实库含)

## 🔄 自我进化接口
每通对话记录"成功/失败 + 卡在哪阶段" → 回流诊断数据 → 监控 agent 优化话术。
