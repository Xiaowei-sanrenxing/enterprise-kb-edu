# 任务卡 · 续费复购转介绍（renewal-referral）

> Agent 操作总台。接到"续班推进/复购/老带新/口碑裂变"类任务，从这里开始。

## 🎯 目标(Goal)
把已交付的学员，转化为续费/复购 + 主动转介绍——**让交付成果变成下一轮获客的起点**（老带新成本远低于公域获客）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/续费节点打法.md`](context/续费节点打法.md) — 在学员"体验到价值"的节点推续费，不是临到期才催
2. [`context/转介绍裂变机制.md`](context/转介绍裂变机制.md) — 老带新的触发点、激励、话术
3. 〔学员成果记录〕 — 客户补：来自交付工作流（脱敏）
4. 〔续费/转介绍激励政策〕 — 客户补：当期政策

## 🪜 指令(Instructions) —— 标准动作流
1. **识别价值节点**：学员拿到成果/做出效果的时刻 = 最佳续费/转介绍窗口
2. **续费**：在价值节点而非到期日推进，用成果说话，不靠催
3. **复购**：根据学员阶段推荐进阶/关联课程
4. **转介绍**：在学员满意峰值发起老带新，给清晰激励 + 现成转介绍话术/物料
5. **闭环**：转介绍来的新线索 → 交回销转工作流（高信任度，转化率更高）
6. 学员成果引用对外 → 必脱敏（走交付工作流的脱敏铁律）

## 🛠 工具(Tools)
〔续费提醒（按价值节点）〕 · 〔转介绍激励系统〕 · 成果记录检索 · 转介绍话术库

## 🔒 权限(Permission)
- 可读：续费政策、转介绍话术、（脱敏的）成果记录
- 🔴 受限：学员个人信息/联系方式
- 需人工确认：超标准的续费优惠、转介绍激励发放

## ✅ 验收(Acceptance)
- 续费在价值节点推进（非到期催）｜ 转介绍在满意峰值发起 ｜ 老带新线索回流销转
- 详见 evals.md

## 🔄 自我进化接口
记录"哪个价值节点续费/转介绍成功率最高" → 回流续费节点打法 → 优化触达时机。
