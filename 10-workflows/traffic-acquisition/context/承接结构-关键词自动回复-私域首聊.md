# 承接结构：关键词、自动回复、私域首聊、转人工

> 这一页不是单一话术，而是承接结构。
> 先分清用户发来的触发词、系统回的第一句、进入私域后的真人首聊，再谈执行。

## 1. 用户触发关键词

用户主动在评论区或私信里发出的领取词、触发词。

示例：

- “求资料”
- “求清单”
- “求课程”
- “报行业”
- “想了解”

用途：

- 判断用户是被哪个钩子吸引进来的
- 决定应该返回哪份资料或哪种回复
- 为后续私域首聊补上下文

## 2. 自动回复内容

系统或人工看到关键词后，第一时间兑现承诺并完成基础承接。

自动回复至少包含：

- 告知资料已发送或领取方式
- 重复一次钩子承诺，避免用户落差
- 给出下一步动作，如“补行业”“加微领取”“填表出分”

自动回复的目的：

- 兑现承诺
- 低人力承接
- 不在这一层做深度销售追问

## 3. 私域首聊话术

用户加上微信或进入私域后的第一句真人话术。

私域首聊的目的：

- 判断对方是泛流量还是真目标客户
- 收集业务类型、具体问题、当前阶段、决策权等信息
- 为后续进入 `sales-conversion` 工作流做准备

私域首聊不等于自动回复。自动回复重在兑现承诺，私域首聊重在继续提纯和判断下一步。

## 4. 转人工条件

满足以下条件时，从自动承接切到人工承接：

- 用户明确表达购买意向
- 用户提出复杂业务问题
- 用户要求 1v1 诊断、报价或方案
- 用户不属于标准自动回复覆盖范围

## 推荐填写格式

客户补这部分时，建议分成四栏：

1. 用户触发关键词
2. 自动回复内容
3. 私域首聊话术
4. 转人工条件

## 常见误区

- 把“用户发来的词”和“我们发给用户的话”写在一起
- 在自动回复阶段就塞太多销售追问
- 没有定义何时应切到人工承接
