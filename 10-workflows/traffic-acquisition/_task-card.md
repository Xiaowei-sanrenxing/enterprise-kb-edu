# 任务卡 · 获客引流（traffic-acquisition）

> Agent 操作总台。接到"设计钩子/写引流内容/承接评论私信/搭漏斗"类任务，从这里开始。

## 🎯 目标(Goal)
把公域流量（视频号/抖音/小红书/B站等）精准引到私域或到店，**提纯**而非泛拉——拉来的是真目标客户，不是白嫖资料粉。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/三层漏斗引流打法.md`](context/三层漏斗引流打法.md) — 顶层放量/中层提纯/底层成交，别用一个钩子同时拉量和筛质
2. [`context/钩子设计公式.md`](context/钩子设计公式.md) — 钩子精准度 ≈ 摩擦门槛 × 行业专属性
3. 〔诱饵库索引〕 — 客户补：现成钩子/诱饵清单
4. [`context/承接结构-关键词自动回复-私域首聊.md`](context/承接结构-关键词自动回复-私域首聊.md) — 区分用户触发关键词、自动回复、私域首聊话术、转人工条件

## 🪜 指令(Instructions) —— 标准动作流
1. 判断这次引流目标是**放量**还是**提纯**，选对应漏斗层（别混）
2. 顶层：设计行业专属轻筛钩子（行业清单/自测表），评论区"报行业发对应那份"→关键词自动回复承接，0 人力
3. 中层：用"填表/付费"摩擦滤掉白嫖（自测表自动出分 / 低价体验课）
4. 底层：限额高精准 1v1（如 AI 落地诊断，每月限名额），人亲自承接
5. 内容产出走双区：新稿进 `_草稿区/`，成熟由负责人提名进 `定版/`

## 🛠 工具(Tools)
〔关键词自动回复系统〕 · 诱饵库检索 · 〔私域接待工具〕 · 〔自测表/出分工具〕 · 内容排期表

## 🔒 权限(Permission)
- 可读：钩子库、诱饵库、关键词触发词、自动回复、私域首聊话术、漏斗数据
- 🔴 受限：引流来的客户名单/联系方式 → 不得写入对外产物
- 需人工确认：1v1 限额名额释放、付费钩子定价改动

## ✅ 验收(Acceptance)
- 钩子有行业专属性（不是"领通用资料"）｜ 有摩擦门槛筛质 ｜ 三层不混用 ｜ 承接自动化
- 详见 evals.md

## 🔄 自我进化接口
每个钩子记录"曝光→加粉→提纯率" → 回流诱饵库 → 监控 agent 淘汰低提纯钩子。
